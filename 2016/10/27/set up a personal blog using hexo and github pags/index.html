title: 采用Hexo和Github Pages搭建个人博客
date: 2016-10-27 22:22:24

没错，这个博客就是这么搭建的。

安装Hexo

安装Hexo步骤很简单，但在之前你需要先安装

  • Node.js
  • Git

安装 Node.js

$ sudo apt-get install nodejs

安装并配置Git

  • 安装git并且配置用户github信息
$ sudo apt-get install git
$ git config --global user.name "<your github user name>"
$ git config --global user.email "<your github email address>"



[帮助文档Generating a new SSH key and adding it to the ssh-agent](https://help.github.com/articles/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent/)

### 安装Hexo

安装完Node.js和Git后，接下来用npm来安装Hexo和Server。后续你可能还需安装一些其他插件。

$ sudo apt-get install npm
$ npm install -g hexo-cli
$ npm install hexo-server –save


## 初始化Hexo并进行测试

* 初始化Hexo

选择一个文件夹来安放Hexo的相关文件，执行以下命令，来初始化Hexo。命令结束后，可以发现这个目录下生成了一些基本的文件。

$ hexo init
$ cd
$ npm install


* 使用默认主题来测试. 

注意到在上一步init的时候，Hexo会默认下载并采用主题landscape。可以`themes`文件夹下找到该主题。接下来执行下面命令，来测试一下。开启Server后，默认可用http://localhost:4000访问.

$ hexo server


## 选择喜欢的博客主题

为了快速搭建博客，我们可以直接使用已有的博客主题(theme)，然后慢慢将其个性化（修修补补一天又一天，哈哈）。留点时间和力气吃肉嘛。
* 在[已有的博客主题](https://hexo.io/themes)中选择一个喜欢的，在github中找到它，fork一下。然后clone到`themes`文件夹。

$ cd
$ git clone https://github.com/jiajiadebug/hexo-theme-yelee.git themes/yelee

* 启用这个主题。在配置文件_config.yml中找到`theme: XXXX`，修改成你选择的主题。我这里是yelee。（注意：Hexo有两个_config.yml文件，一个在根目录，一个在theme下，此时修改的是在根目录下的。）

$ cd
$ sudo vim _config.yml # 修改theme: yelee


* 重启Server，刷新一下浏览器。现在这个博客上采用的就是刚刚你选择的主题。

ctrl + c
$ hexo server

啦啦啦，现在可以修改主题，将其个性化。比如改图片，放点萌照啦。。。压抑住情绪，继续部署。。。


## 将博客部署到github上
Hexo提供了快捷的部署方案。几乎是一条命令即可。在首次部署的时候，需要配置一些信息。

* 在github上创建一个repository: `username.github.io`。了解更多关于[github page信息](https://pages.github.com/)
* 安装hexo-deployer-git

$ npm install hexo-deployer-git –save

* 先修改根目录下的_config.yml，将其指向刚刚创建的repository。

deploy:
type: git
repo: git@github.com:jiajiadebug/jiajiadebug.github.io.git
branch: master


* 用generate命令生成静态文件。了解更多关于[generate信息](https://hexo.io/zh-cn/docs/generating.html)。

$ hexo generate


可以用以下命令来检查文件变动情况。

$ hexo generate –watch

可以用以下命令在生成文件后进行部署。这两个命令功能是一样的。

$ hexo generate –deploy
$ hexo deploy –generate


* 将博客部署到github上。
可以用以下命令在生成文件后进行部署。这两个命令功能是一样的。

$ hexo generate –deploy
$ hexo deploy –generate
```

  • 访问http://username.github.io。哒啦啦。至此，已将博客部署到github上，并可以通过http://username.github.io来访问了。

了解更多deploy信息。

注册域名绑定博客

  • 购买域名

我是通过aliyun购买的域名，购买过程挺简单。首先查询一下想注册的域名是否被注册，然后加入订单，选择年数，支付宝付款即可。

  • 解析

添加如下规则到解析列表中。启动它。然后等待。

CNAME @ username.github.io.

  • 状态

吐槽一下，没有实名，购买五天后，被hold住了，没有任何通知。域名实名一提交，虽然还在待审核，但是域名马上可以用了。建议装阿里云到手机，可以及时知道域名状态。

更新主题和博文

  • 更新主题

推荐先enable那些自己觉得最不可或缺的功能或模块。小细节的东西可以在以后慢慢更新。

  • 更新博文

可以在命令中指定文章的布局（layout），默认为post，可以通过修改_config.yml中的default_layout参数来指定默认布局。Hexo 有三种默认布局：post、page 和 draft，它们分别对应不同的路径(source/_posts, source, source/_drafts)。

也可以完全手动添加删除这些folder下面的文件。

  • 推荐Markdown工具

MacDown不错，我在用。
